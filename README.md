# 新标签页 · 星图

Chrome 浏览器扩展（Manifest V3），替换新标签页，呈现一个**星图（Constellation）** 视觉：

- 中心显示时钟与今日摘要
- 所有书签按分组放置在四个象限（工作/娱乐/阅读/工具 等）
- 节点大小与访问频次成正比，距离中心越近代表越常用
- 悬停展示名称/域名，点击在新标签页打开
- 右键节点或分组标签呼出菜单

设计稿来自 Claude Design 原型（`design/`，4 选 1 已选定星图风格），迁移为 React + TypeScript + Vite 项目。

## 功能

### 书签 CRUD（全部接入 `chrome.bookmarks`）

| 操作 | 入口 |
| ---- | ---- |
| 新建书签 | 顶部工具栏 **＋ 添加** |
| 新建分组（文件夹） | 顶部工具栏 **＋ 新建分组** |
| 打开 | 点击节点（新标签页打开） |
| 编辑（改名 / 改网址 / 改分组） | 右键节点 → 编辑书签 |
| 删除书签 | 右键节点 → 删除书签 |
| 重命名分组 | 右键分组标签 → 重命名 |
| 删除分组 | 右键分组标签 → 删除（需先清空） |

### 任务（Todos）

顶部 **今天** 按钮切出侧边面板；支持添加、完成、删除；通过 `chrome.storage.local` 持久化，按钮旁的小徽章显示未完成数量。

### 过滤

顶部输入框或按 `/` 聚焦；输入后不匹配的节点会淡出，匹配唯一一项时 Enter 直接打开。

### 外观

右下齿轮按钮：深色/浅色主题、6 种色调、宽松/紧凑密度、纯色/颗粒/网格背景。全部通过 `chrome.storage.local` 持久化。

## 数据来源（真实 Chrome API）

- `chrome.bookmarks` — 书签栏第一层文件夹映射为"分组"；散落的书签归入"置顶"
- `chrome.history` — 近 30 天历史聚合访问频次 + 最近访问列表
- `chrome.topSites` — 书签为空时的兜底数据源
- `chrome.storage.local` — 偏好设置 + 任务持久化
- `https://{domain}/favicon.ico` — 站点原生图标（未用在星图节点上，保留给 favicon 字母色块映射）

书签在 Chrome 里被外部修改（书签管理器、同步、拖拽）也会通过 `chrome.bookmarks` 事件自动反映到页面。

## 快捷键

| 按键 | 作用 |
| ---- | ---- |
| `/` | 聚焦过滤框 |
| `Enter` | 打开唯一匹配项 |
| `Esc` | 关闭对话框或任务面板 |

## 开发

```bash
npm install
npm run dev
```

打开 <http://localhost:5173>。**注意**：Dev 模式下没有 Chrome 扩展上下文，`chrome.bookmarks` 等 API 不可用，页面会使用 `src/data/fallback.ts` 的 mock 数据，CRUD 操作为无效。

## 构建并安装到 Chrome

```bash
npm run build
```

1. 打开 `chrome://extensions`
2. 右上角开启 **开发者模式**
3. 点击 **加载已解压的扩展程序** → 选择 `dist/` 目录
4. 打开新标签页即可

## 权限说明

| 权限 | 用途 |
| ---- | ---- |
| `bookmarks` | 读写书签（增删改查，分组管理）|
| `history` | 聚合访问频次 + 最近访问 |
| `topSites` | 书签为空时的兜底数据 |
| `storage` | 偏好设置 + 任务持久化 |
